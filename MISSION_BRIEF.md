# MISSION BRIEF: Sprint 1 (Data Integration & Backend Bring-up)

## 🎯 Objective
Replace the static placeholder data generated by v0 with live, dynamically calculated reliability scores from our 10-year eBird database.

## 🤖 Agent Task List

### Phase 1: Backend API Development (Backend Agent)
- [x] **Task 1: Setup.** Initialize a FastAPI application in `/backend`. Create a `requirements.txt` that includes `fastapi`, `uvicorn`, `supabase`.
- [x] **Task 2: The Endpoint.** Create a `GET /api/predictions` endpoint. It must accept: `latitude` (float), `longitude` (float), `month` (int, starts at 1), `radius_miles` (float, default 10), and `years_limit` (int).
- [x] **Task 3: Database Call.** Inside the endpoint, call the Supabase RPC `get_seasonal_hotspots(user_lat, user_lon, month, radius, years)`.
- [x] **Task 4: Response Formatting.** Ensure the JSON response perfectly matches the TypeScript interface expected by `results-list.tsx`.

### Phase 2: Frontend Data Hookup (Frontend Agent)
- [x] **Task 1: The API Hook.** In `frontend/hooks/`, create a new file called `use-predictions.ts`. Write a custom hook to fetch data from `http://localhost:8000/api/predictions` passing the form parameters.
- [x] **Task 2: Form Wiring.** Update `frontend/components/prediction-form.tsx` to capture the user's latitude, longitude, month, and year-range, and pass these to the fetch hook.
- [x] **Task 3: UI Integration.** Update `frontend/components/results-list.tsx` to consume the live API data instead of the data from `frontend/lib/mock-sightings.ts`.
- [x] **Task 4: Error Handling.** Use the existing Shadcn `alert.tsx` or `use-toast.ts` to show an error if the FastAPI backend is down or returns 0 results.

### Phase 3: Backend API Development For Annual Summaries (Backend Agent)
- [x] **Task 1: The 2nd Endpoint.** Create a `GET /api/stats/annualsummary` endpoint. It does not require any parameter.
- [x] **Task 2: Database Call.** Inside the endpoint above, call the Supabase RPC `get_annual_sightings_summary()`.
- [x] **Task 3: The 3rd Endpoint.** Create a `GET /api/stats/state` endpoint. It must accept: `year` (int) as a parameter.
- [x] **Task 4: Database Call.** Inside the endpoint above, call the Supabase RPC `get_state_stats_by_year()`.
- [x] **Task 5: The 4th Endpoint.** Create a `GET /api/stats/counties` endpoint. It must accept: `state` (text) and `year` (int) as parameters.
- [x] **Task 6: Database Call.** Inside the endpoint above, call the Supabase RPC `get_county_stats_by_state_year()`.

### Phase 4: Data Visualization & Interactive Stats Dashboard (Frontend Agent)
- [x] **Task 1: Global Navigation.** Add a top-level navigation menu or tab system to the main layout (`frontend/app/layout.tsx` or main page) to allow the user to navigate between the "Predictions" (Home) and the new "Stats" (`/stats`) page.
- [x] **Task 2: API Hooks Setup.** In `frontend/hooks/`, create new data fetching hooks (`use-stats.ts` or similar) that communicate with the backend:
    - `GET /api/stats/annualsummary`
    - `GET /api/stats/state?year={year}`
    - `GET /api/stats/counties?state={state}&year={year}`
- [x] **Task 3: Stats Page Routing & State.** Create the `frontend/app/stats/page.tsx`. Use Next.js URL Search Parameters (`?year=...&state=...`) to track the current drill-down level. This ensures browser history and "Back" buttons work perfectly.
- [x] **Task 4: Annual Summary View (Level 1).** If no URL parameters exist, fetch the annual summary. Render a bar chart (using Shadcn UI Charts / Recharts) showing counts per year. Make the bars clickable. Clicking a bar updates the URL to `?year={selected_year}`.
- [x] **Task 5: State Drill-Down View (Level 2).** If the `year` parameter exists but `state` does not, fetch the state-level data for that year. Render a bar chart showing sightings by state. Include a Shadcn UI "Back" button that clears the `year` parameter. Make the bars clickable to update the URL to `?year={selected_year}&state={selected_state}`.
- [x] **Task 6: County Drill-Down View (Level 3).** If both `year` and `state` parameters exist, fetch the county-level data. Render a bar chart showing sightings by county. Include a "Back" button that clears only the `state` parameter to return to Level 2.
- [x] **Task 7: Styling & Theming Consistency.** Ensure all charts, cards, buttons, and backgrounds strictly inherit the existing Tailwind CSS variables, dark/light mode configurations, and structural design used in `prediction-form.tsx` and `results-list.tsx`. Do not introduce new color palettes.
